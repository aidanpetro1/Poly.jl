**One ask-correction during the round.** Aidan's original sketch for
`weak_dual` had "positions are functions p(1) → p(1)" which is wrong;
the actual FA Example 7.2 formula is `p^∨ = Σ_{f ∈ Π p[i]} y^{p(1)}`

`internal_hom(p, y)` exactly. The docstring's "Caveats" section
retracts the sketch.

## What's still pending (v0.7)

None of these have a design doc yet. Order is roughly increasing
implementation cost:

1. **Symbolic / NatSet path for `coclosure`** — makes
   `comonoid_from_list_polynomial(list_polynomial())` work without
   truncation (full Fin^op rather than truncated).
2. **Multi-variable Dirichlet ⊗ on `d-Set[c]`** (FA §7.2-7.3, Def 7.11,
   Lemma 7.13, Cor 7.15). Introduces `d-Set[c]` as a first-class
   category in the codebase. Load-bearing for everything below.
3. **Theorem 7.19 contravariant equivalence** `Cat#_lin ↔ Cat#_disc,con`
   via weak duality.
4. **Profunctor ↔ conjunctive bicomodule dictionary** (FA Prop 7.25,
   Prop 7.27).
5. **Generalize `kan_along_bicomodule`** to general `(c, d)`
   (FA Prop 6.7).
6. **Duoidality natural map**
   `(p ◁ q) ⊗ (p' ◁ q') → (p ⊗ p') ◁ (q ⊗ q')`
   (FA Prop 7.10, 7.17).
7. **Full `(B, E, π, S, T)` bridge diagram** for general bicomodules
   (FA Prop 3.20). Replaces / sits alongside v0.6's linear-case
   projection.

Plus the v0.6 carryovers that were never blocking PolyAggregation
v0.3.0 (and so were dropped from the v0.6 ship):

- `is_conjunctive` + `ConjunctiveBicomodule(B)` (Cor 6.15).
- `Span{A,B}` struct + finite-set pullback in
  `compose(::Span, ::Span)` + convenience
  `linear_bicomodule_from_span(span)`.
- Lift `linear_bicomodule_from_span` to non-discrete bases via the
  bridge-derived right action.
- `c_one_y` c-c packaging variant (the v0.5.1 watchpoint).
